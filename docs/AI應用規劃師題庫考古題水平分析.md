# AI應用規劃師題庫與考古題水平分析

## 結論

現行主題庫在「章節覆蓋」與「核心術語覆蓋」上已接近考古題，但主題庫本身尚未完全達到考古題同等水平。主要差距不在題數或考點，而在題目品質：官方考古題更精準、干擾選項更貼近、實作判讀題更多，且部分題目要求計算、程式碼、圖表或多敘述判斷。

因此目前採用雙層題庫設計：原兩科各 1000 題保留作為大量概念練習，另新增「考古題與高擬真補強」科目，專門承接正式考試風格訓練。

## 已完成補強

- 第二科：Z-Score、Gini impurity、pandas/seaborn/sklearn 語法、groupby、astype、isna、迴歸與視覺化判讀。

## 已達標的部分

- 章節配比合理：兩科都已依考古題趨勢與學習指引篇幅加權。
- 核心術語覆蓋完整：NLP、Transformer、BERT、Word2Vec、TF-IDF、IoU、mAP、DBSCAN、PCA、ACID、Robust Scaling、同態加密等都有進入題庫。
- 情境式題型足夠：現行題庫大量使用企業、資料流程、模型導入與治理情境。
- 單選格式穩定：每題皆有四選項、答案與解析，適合練習系統使用。

## 尚未達標的部分

1. 官方題目更常是「單一明確考點」

考古題通常先給一個場景，再直接問某技術的主要目的、核心限制、正確定義或最適方法。現行題庫常把多個背景條件混在一起，例如同一題同時出現資料來源、法遵、模型限制、驗收重點，反而降低題目銳利度。

2. 科目二缺少真正的實作判讀

考古題科目二有明確的 pandas、seaborn、LinearRegression、groupby、astype、isna 等語法題，也有附圖/下圖判讀題。現行題庫雖出現這些字串，但多半只是背景文字，沒有真正要求考生判斷語法正誤、輸出結果或圖表意義。

3. 計算題比例明顯不足

考古題科目二包含 Z-Score、Gini impurity、分位數、缺失值統計、迴歸係數判讀等。現行題庫的數字、公式、計算類題目比例只有約 4.8%，不足以模擬正式考試壓力。

4. 英文術語密度不足

考古題高度使用中英並列，例如 `Bidirectional Encoder Representations from Transformers`、`Masked Language Model`、`Cumulative Distribution Function`。現行題庫雖已補入縮寫，但英文全名、縮寫與中文概念之間的互相辨識題仍偏少。

5. 干擾選項還不夠像考古題

官方選項常是同章節相近概念間的細微差異，例如 Softmax vs Max-Pooling、Normalization vs Standardization、Feature Selection vs Feature Derivation。現行題庫部分選項仍偏定義式，錯誤選項較容易排除。

## 建議調整方向

- 保留現行 2000 題作為基礎練習題庫。
- 另外建立每科 100 至 200 題「考古題風格高擬真題」。
- 科目一加強：中英術語辨識、相近模型比較、導入治理情境中的最佳作法題。
- 科目二加強：pandas/seaborn/sklearn 語法題、圖表判讀題、Z-Score/Gini/分位數/迴歸輸出計算題。
- 題幹改寫方向：減少無關背景詞，讓每題只鎖定一個主考點。
- 干擾選項改寫方向：用同章相近概念做干擾，而不是明顯跨域錯誤。
