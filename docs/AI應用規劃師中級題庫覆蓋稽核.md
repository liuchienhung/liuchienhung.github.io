# AI應用規劃師(中級)題庫覆蓋稽核

## 稽核結論

AI 應用規劃師(中級)題庫已拆成兩科，各 1000 題單選題。題數分配不再平均切分，而是採用混合權重：

- 70%：依 114 年第二梯次公告試題的章節落點估算出題趨勢。
- 30%：依學習指引章節篇幅與內容量保留完整覆蓋。

此配置的目的不是複製公告試題，而是讓高頻考點加權，同時避免低頻但仍在學習指引內的章節被排除。

## 覆蓋檢查

目前每個單元至少有 16 個概念種子進入題目設計。概念種子包含原章節核心名詞，也補入學習指引常見細項能力點，例如資料流程、模型評估、風險治理、部署維運、統計推論、資料處理工具、可視化設計與隱私合規。

題目章節一致性採兩層檢查：

- 題幹主考點：每題題幹中的「考點」必須來自該單元的概念種子。
- 選項干擾項：錯誤選項不再使用跨章明顯錯誤概念，而是從同一單元的其他概念種子抽取，讓干擾項仍落在該章節範圍內。
- 題型語氣：科目一採「企業情境 + 指定 AI 技術/模型 + 問主要目的、核心作用或功能」；科目二採「資料整合/分析/治理情境 + 條件約束 + 問最合適方法或設計」。
- 情境資料：資料來源會依章節挑選，例如 NLP 題使用客服、文本、評論或文件資料；電腦視覺題使用影像、標註框、OCR 或關鍵點資料；大數據題使用統計量、資料管線、知識圖譜、儀表板、隱私紀錄等對應資料。

例如自然語言處理題會以 BERT、Transformer、TF-IDF、斷詞、語意相似度等同章概念互為干擾；電腦視覺題會以 IoU、mAP、CNN、NMS、OCR、遷移學習等同章概念互為干擾；不會再用資料庫交易或機械手臂這類跨章概念作為主要干擾項。

執行檢查：

```bash
node tools/generate-ai-planner-questions.js
node tools/audit-ai-planner-coverage.js
```

稽核通過條件：

- 第一科合計 1000 題。
- 第二科合計 1000 題。
- 每單元題數符合 70% 考古題趨勢 + 30% 指引篇幅的分配。
- 每單元題幹皆唯一。
- 每單元選項組合皆唯一。
- 每單元偵測到 16 個概念種子。

## 注意

114 年第二梯次公告試題只有一份樣本，不能視為官方固定配比。本題庫採用該份公告試題作為趨勢參考，並用學習指引篇幅補足低頻章節的題量，避免過度擬合單一梯次。
