# 國小國語試卷報讀服務需求規劃書

## 1. 文件目的

本文件定義一項新的網站服務:「國小國語試卷報讀服務」。

服務目標是讓使用者上傳國小國語考試試卷 PDF 或圖片後，系統能自動辨識試卷版面與題型，建立可互動的報讀介面。使用者可先點選題目聆聽題幹，再點選各選項逐一聆聽內容，特別強化選擇題的操作體驗。

本規劃以樣本試卷 `C:\Users\neilliu\Downloads\113-2-2 國語科試卷.pdf` 為需求依據之一。

## 2. 背景與問題

國小國語試卷常見以下特性:

- 採直排中文版面。
- 同一份試卷混合多種題型，例如寫國字或注音、改錯字、連連看、選擇題、照樣寫短語、造句、閱讀測驗。
- 題目編號、配分、說明文字、題幹、選項與作答空格散布在不同位置。
- 選擇題和閱讀測驗通常是最適合先做互動報讀的題型，因為題幹與選項結構相對穩定。

目前網站已有「多服務入口 + 題庫測驗服務」，但尚未提供上傳真實考卷並進行朗讀的服務。

## 3. 服務定位

### 3.1 服務名稱

國小國語試卷報讀服務

### 3.2 服務定位

這是一個面向國小學童、教師、家長與陪讀者的輔助工具，主要用途如下:

- 協助閱讀能力較弱或需要語音輔助的學生理解試卷內容。
- 降低直排中文試卷在數位閱讀上的理解門檻。
- 將靜態試卷轉為可點擊、可逐題、可逐選項聆聽的互動頁面。

### 3.3 首波範圍

MVP 先聚焦在:

- 上傳 PDF 或圖片檔。
- 自動分析頁面與題型。
- 顯示可點擊題目區塊。
- 支援題目朗讀。
- 支援選擇題選項朗讀。
- 支援閱讀測驗題組的題幹與子題朗讀。

不在 MVP 內的項目:

- 自動批改。
- 自動辨識手寫作答內容。
- 所有非選擇題的完整互動作答。
- 帳號系統與雲端歷史紀錄。

## 4. 樣本試卷分析摘要

依樣本試卷觀察，至少需要支援以下題型辨識:

- 寫國字或注音。
- 改錯字。
- 連連看。
- 選擇題。
- 照樣寫短語。
- 造句。
- 閱讀測驗。

其中選擇題有兩種主要情境:

- 一般單題選擇題。
- 閱讀測驗題組下的子題選擇題。

版面風險如下:

- 直排文字與注音混排。
- 選項符號可能不是 `A/B/C/D`，而是 `①②③④`、`ロヮワヰ` 等。
- 題號、配分與題幹之間可能分行或跨欄。
- 題組文章與子題可能分頁。

因此本服務不能只靠單純抽文字，必須結合版面區塊分析。

## 5. 目標使用者

### 5.1 學生

- 需要逐題聆聽內容。
- 希望點一下題目就能聽題幹。
- 點選選項時，希望聽到選項代號與內容。

### 5.2 教師或家長

- 上傳試卷後，希望快速檢查辨識是否正確。
- 需要在必要時修正題型或題目切分。

### 5.3 陪讀或特教輔助情境

- 需要明確、穩定、重複播放的朗讀互動。
- 需要降低誤觸與誤讀。

## 6. 核心使用流程

### 6.1 上傳與分析

1. 使用者進入服務首頁。
2. 上傳試卷 PDF、JPG 或 PNG。
3. 系統顯示上傳中與分析中狀態。
4. 系統完成頁面切圖、OCR、版面分析與題型判斷。
5. 系統產生可互動的試卷閱讀畫面。

### 6.2 題目報讀

1. 使用者點選某一題。
2. 系統高亮該題區塊。
3. 系統朗讀:
   - 題號。
   - 題型。
   - 題幹內容。
4. 若該題為閱讀測驗子題，可視需要先朗讀題組文章摘要或全文。

### 6.3 選項報讀

1. 使用者在選擇題中點選某個選項。
2. 系統只朗讀該選項，不自動切換到下一題。
3. 朗讀格式建議為:
   - `第 4 題，選項 2，...`
   - 或 `第 4 題，ヮ，...`
4. 使用者再次點選同一選項時，可重新播放。

## 7. 功能需求

## 7.1 檔案上傳

- 支援 PDF、PNG、JPG、JPEG。
- 顯示檔名、頁數、上傳進度與分析狀態。
- 限制檔案大小與頁數，避免前端與伺服器負擔過重。
- 若上傳失敗或格式不支援，須提供明確錯誤訊息。

## 7.2 試卷分析

- 將 PDF 轉為逐頁影像。
- 進行 OCR 與文字區塊定位。
- 辨識直排中文版面。
- 分析區塊層級:
  - 頁面。
  - 大題。
  - 小題。
  - 題幹。
  - 選項。
  - 題組文章。
- 自動判斷題型，至少包含:
  - 選擇題。
  - 閱讀測驗。
  - 填空或書寫題。
  - 配對題。
- 對低信心判斷保留人工修正能力。

## 7.3 互動閱讀畫面

- 顯示試卷縮圖或頁面畫布。
- 題目區塊需可點擊。
- 被選取的題目需高亮。
- 顯示目前頁數、目前題號、目前題型。
- 支援上一頁、下一頁。

## 7.4 朗讀功能

- 點題目時朗讀題幹。
- 點選項時朗讀該選項。
- 提供播放、暫停、重播。
- 朗讀時同步標示正在播放的題目或選項。
- 支援調整語速。
- 預設使用繁體中文語音。

## 7.5 選擇題專屬互動

- 系統需將選擇題拆成:
  - 題號。
  - 題幹。
  - 選項代號。
  - 選項內容。
- 點選題目本體:
  - 朗讀題號與題幹。
  - 不自動連續朗讀所有選項，避免資訊過載。
- 點選選項:
  - 朗讀該選項代號與內容。
  - 例如 `選項一...`、`ロ...`
- 提供「朗讀全部選項」按鈕作為輔助功能，但不作為預設互動。

## 7.6 閱讀測驗題組

- 辨識題組文章與其對應子題。
- 使用者可先點文章區塊朗讀文章。
- 點子題時朗讀子題題幹。
- 點子題選項時朗讀選項。
- 題組文章跨頁時，仍要維持文章與子題的關聯。

## 7.7 人工修正

為了提高實用性，需預留教師或管理者修正能力:

- 修正大題範圍。
- 修正小題切分。
- 修正題型。
- 修正選項切分。
- 修正 OCR 文字。

此功能可列為 MVP 後的 Phase 2，但資料模型需先保留。

## 8. 非功能需求

## 8.1 正確性

- 選擇題題型辨識正確率應優先達標。
- 題幹與選項切分應可被人工驗證。
- OCR 錯字不應直接導致題型判斷失敗。

## 8.2 效能

- 一份 2 到 4 頁的國小試卷，目標在合理時間內完成分析。
- 首頁需清楚顯示目前分析狀態，避免使用者誤以為系統卡住。

## 8.3 易用性

- 操作要適合國小學生。
- 按鈕尺寸要足夠大，避免誤觸。
- 播放中的題目與選項要有明顯視覺回饋。

## 8.4 可維護性

- 題型分類規則與朗讀流程應模組化。
- 前端服務入口需能再新增其他教育服務。
- 試卷分析結果應以結構化 JSON 儲存，便於除錯與後續再利用。

## 10. 系統模組建議

### 10.1 前端

- 服務首頁入口。
- 檔案上傳介面。
- 分析結果畫面。
- 題目互動與播放控制。

### 10.2 後端或分析服務

- PDF 轉圖。
- OCR。
- 版面分析。
- 題型分類。
- 結構化資料輸出。
- TTS 音訊生成或即時語音串接。

### 10.3 管理修正工具

- 題目框選與調整。
- 題型覆寫。
- OCR 文字修正。

## 11. MVP 驗收條件

- 可上傳一份國小國語試卷 PDF。
- 系統可完成頁面分析並產生可點擊畫面。
- 系統可正確辨識選擇題區塊與至少 80% 以上的選項切分。
- 使用者點題目時可朗讀題幹。
- 使用者點選項時可朗讀該選項。
- 閱讀測驗題組至少可支援文章區塊與子題的基本關聯。
- 若辨識失敗，系統能清楚提示哪些題目需要人工修正。

## 12. 風險與注意事項

- 直排中文加注音會提高 OCR 與題目切分難度。
- 不同學校試卷格式差異大，規則不能只針對單一版型。
- 若只做文字抽取，會失去題目與選項的空間位置，影響點擊互動。
- 朗讀體驗若一次讀太多內容，學生不容易吸收，因此預設互動應以「點題目讀題幹、點選項讀選項」為主。

## 13. 建議開發階段

### Phase 1

- 完成服務入口與上傳流程。
- 支援 PDF 轉圖與基礎 OCR。
- 先做好選擇題與閱讀測驗的題型辨識。
- 完成題目點擊朗讀與選項點擊朗讀。

### Phase 2

- 加入人工修正介面。
- 優化非選擇題的區塊辨識。
- 支援更多試卷格式。

### Phase 3

- 加入歷史紀錄。
- 加入教師批次匯入與管理。
- 加入多語音、語速與朗讀模式設定。

## 14. 本文件產出結論

本服務的 MVP 不應一開始追求「所有題型都能完整互動作答」，而應優先建立一條可靠流程:

- 上傳試卷。
- 自動判斷題型。
- 正確切出選擇題與閱讀測驗。
- 使用者點題目可聽題幹。
- 使用者點選項可聽選項。

在這個基礎上，再逐步補強人工修正與更多題型支援，會比一次做完整自動判斷更可控，也更符合樣本試卷的真實複雜度。
